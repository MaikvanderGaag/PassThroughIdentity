# Introduction

In situations you need to login to an application and use that identity to access an API (pass-through identity) and also get data from Azure SQL Server. With the new feature to use Azure Active Directory users within Azure SQL Server this is know is possible.
In this series of blog post I will cover the following topics:

- [Part 1 – Azure SQL Database with Azure Active Directory Authentication](https://msftplayground.com/2017/04/part-1-azure-sql-database-with-azure-active-directory-authentication)
- [Part 2 – Azure API Application to query the Azure SQL Database](https://msftplayground.com/2017/04/part-2-azure-api-application-to-query-the-azure-sql-database)
- [Part 3 – Console application to call the API](https://msftplayground.com/2017/04/part-3-console-application-to-call-the-api)


The source code is the result of the series of blog posts described above:
[Pass-Through Authentication with Azure Active Directory, Azure SQL, Azure API and a Console Application](https://msftplayground.com/2017/04/pass-through-authentication-with-azure-active-directory-azure-sql-and-azure-api-application/)
